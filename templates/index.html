GSTR-3B Consolidated Report Structure

  • Comprehensive Consolidation: Combines multiple monthly GSTR-3B JSON files into a single Excel workbook with two sheets: Meta_Data and GSTR-3B.
  • Exact Format Preservation: The output matches the official GSTR-3B return format, including all section headers and row labels exactly as specified by GSTN.
  • Automatic Calculations: Net ITC, total tax payable, and column-wise totals are computed automatically using built-in formulas.
  • Multi‑Month View: Each month's data appears in a separate column, with months sorted chronologically for easy comparison.
  • Zero‑Fill for Missing Values: Any field not present in a particular JSON is filled with zero, ensuring complete tables.
  • Header Merging: Section headers are merged across columns to maintain the original GSTR-3B appearance.
  • Professional Styling: Dark blue headers, light blue section backgrounds, and border lines exactly as per the standard format.

    • Important Notes

    This tool is designed for standard GSTR-3B JSON exports from the GST portal. Please verify the consolidated data against original returns before filing.

    Intelligent JSON‑to‑Excel Mapping

    The tool uses a sophisticated mapping engine to accurately place each JSON field into the correct cell of the GSTR-3B sheet. Key mappings include:

    • sup_details.osup_det → Section 3.1(a) – Outward taxable supplies
    • sup_details.osup_zero → Section 3.1(b) – Zero rated supplies
    • sup_details.osup_nil_exmp → Section 3.1(c) – Nil rated / exempted
    • sup_details.isup_rev → Section 3.1(d) – Inward supplies under reverse charge
    • eco_dtls.eco_sup → Section 3.1.1(i) – ECO pays tax u/s 9(5)
    • inter_sup.unreg_details/comp_details/uin_details → Section 3.2 – Inter‑State supplies
    • itc_elg.itc_avl (by type IMPG, IMPS, ISRC, ISD, OTH) → Section 4(A) – ITC Available
    • itc_elg.itc_rev → Section 4(B) – ITC Reversed
    • itc_elg.itc_inelg → Section 4(D)(2) – Ineligible ITC
    • intr_ltfee.intr_details / ltfee_details → Section 5.1 – Interest & Late fee
    • taxpayble.returnsDbCdredList.tax_pay (with trancd 30002/30003) → Section 6.1 – Tax payable
    • taxpayble.returnsDbCdredList.tax_paid.pd_by_cash / pd_by_itc → Section 6.1 – Tax paid in cash / through ITC

    All mappings are fully documented and can be extended to support future JSON schema changes.

    Frequently Asked Questions (FAQs)

    1. What types of Excel files can I merge using this tool?

    You can merge any Excel files such as attendance sheets, salary sheets, registers, bank statements, MIS reports, or custom internal reports. The tool is designed to handle different formats, column orders, and structures, not just identical templates.

    2. Do all Excel files need to have the same columns or format?

    No. Files do not need to follow the same format. The tool intelligently detects headers, aligns similar columns, and merges data logically. However, extremely unstructured files may require manual review after merging.

    3. How does the tool handle different or multi-row headers?

    The tool automatically scans the file to identify the actual data headers, even if there are title rows. Headers span multiple rows. Cells are merged Non-data rows like titles or notes are ignored or handled separately during the merge.

    4. Will the original data in my Excel files be changed?

    No. Your uploaded files are never modified. The tool only creates a new merged Excel file, keeping all original data intact.

    5. What happens if some columns exist in one file but not in others?

    The tool creates a combined structure that includes all detected columns. If a column is missing in a particular file, the corresponding cells in the merged output will remain blank for those rows.

    6. Is this tool suitable for large or professional datasets?

    Yes. The tool is built for real-world business data, including multi-sheet files, varying formats, and large datasets. Still, users should always review the merged output before using it for reporting or compliance purposes.

    GST JSON Download Instructions

    Follow the steps below to generate JSON files and create your consolidated GSTR report:

    1. Use Google Chrome and click the button below to install the required extension.
    2. Click “Add to Chrome” and complete the installation.
    3. Log in to GST Portal → Go to Return Dashboard → Click the 🧩 Extensions icon at the top-right → Select Taxtribute GST Assistant.
    4. Select Financial Year → Choose return type → Click Generate JSON → Download the ZIP file and extract all JSON files.
    5. Return to this tool → Upload extracted JSON files (GSTR-wise) → Click Generate Report to create the consolidated Excel report.
